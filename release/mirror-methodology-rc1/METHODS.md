# Methods

The package was assembled as a clean release-candidate folder rather than by exposing the live MIRROR workspace. The paper draft was rewritten into a compact external-reader structure. Companion files state scope, limitations, data posture, reproducibility steps, and claim boundaries.

The release candidate uses the MIRROR reference bundler from the private workspace to create local evidence bundles for `paper.md` and for a minimal example. Generated bundle metadata was reviewed so public-facing files avoid private absolute workspace paths. Verification remains local and offline.

The package intentionally excludes worker diaries, outgoing sibling-lane tickets, live daily roll-up outputs, private filesystem inventories, and large source snapshots that are not necessary for first human review.
